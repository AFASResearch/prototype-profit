1. Inleiding

1.1 Aanleiding

Profit heeft geen functie voor periodetoekenning van omzet op achteraf-abonnementen. De tabel Toekenningsregels (FbFalAss) bestaat niet en er is geen ingangspunt om journalisatie te starten.

1.2 Resultaat

Een nieuw tabblad Periodetoekenningsregels op het bestaande scherm frmFiPeriodeafsluiting (Periodeafsluitingsplan) met de volgende acties:

  1. Genereer periodetoekenningsregels – maakt FbFalAss-records aan met status Gegenereerd; nog geen journaalposten.
  2. Journaliseer toekenningsregels – boekt journaalposten en zet de status naar Actief.
  3. Journaliseren ongedaan maken – maakt tegenboekingsrecords aan en boekt tegenjournaalposten.

Het tabblad opent met een leeg grid; de gebruiker start de Genereer-actie expliciet.

1.3 Samenvatting voor klant

Deze wijziging zorgt ervoor dat omzet uit achteraf-abonnementen beter en controleerbaar wordt toegerekend aan de juiste financiële periode. Je krijgt hiervoor op het bestaande Periodeafsluitingsplan een extra tabblad met duidelijke acties voor genereren, journaliseren en terugdraaien.

Wat dit je oplevert:

  • Minder handmatige correcties bij periodeafsluiting.
  • Betere financiële controle door een duidelijke audit trail.
  • Snellere verwerking omdat bestaande boekingslogica wordt hergebruikt.
  • Meer zekerheid dat openstaande abonnementenomzet tijdig wordt verwerkt.
  • Consistente afhandeling van correcties, creditfacturen en beëindigingen.

1.3.1 Korte klantsamenvatting voor besluitvorming

Met deze wijziging krijgt je team op een bestaand scherm een praktische werkwijze om omzet uit achteraf-abonnementen per periode te verwerken.

  • Je voorkomt blokkades en handmatige correcties rond periodeafsluiting.
  • Je houdt grip op kwaliteit via duidelijke statussen, controles en audit trail.
  • Je behoudt snelheid door bestaande journalisatie en rechten te hergebruiken.
  • Je beperkt risico bij correcties en verwijderingen door automatisch terugdraaien.

Resultaat: meer voorspelbare periodeafsluiting met minder herstelwerk.

6.6 Toe te kennen regels selecteren

Criteria voor toe te kennen regels:

  • Abonnement is achteraf.
  • Periode is nog niet afgesloten.
  • Geen bestaand FbFalAss-record met Status IN ('Gegenereerd','Actief') voor die jaar/periode-combinatie.
SELECT f.Id
FROM dbo.FbFal AS f
INNER JOIN dbo.FbSul AS s ON s.Id = f.FbSulId
INNER JOIN dbo.FbSub AS b ON b.Id = s.FbSubId
INNER JOIN dbo.AfasFiPeriodClosure AS pc
    ON  pc.AfasFiYear   = @Year
    AND pc.AfasFiPeriod = @Period
    AND pc.IsClosed     = 0
WHERE b.InvoiceMoment = 'Achteraf'
    AND NOT EXISTS (
        SELECT 1
        FROM dbo.FbFalAss AS a
        WHERE a.FbFalId = f.Id
            AND a.AfasFiYear = @Year
            AND a.AfasFiPeriod = @Period
            AND a.Status IN ('Gegenereerd', 'Actief')
    );

6.11 Robuustheid

  • Genereer is herhaalbaar: een tweede uitvoering voor dezelfde periode heeft geen effect.
  • Journaliseer is tolerant: alleen records met Status = Gegenereerd worden verwerkt.
  • Transactioneel per stap: bij fout volledige terugdraaiing.
  • Geen wizard: geen afhankelijkheid van periodeafsluitingsprocedure.
  • Wachtrij/batchservice: alle acties asynchroon; UI blokkeert niet.
  • Concurrency: gefilterde UNIQUE-constraint voorkomt dubbele records.

    Amount       DECIMAL(18, 2)    NOT NULL,
    Status       VARCHAR(20)       NOT NULL
                     CONSTRAINT CK_FbFalAss_Status
                     CHECK (Status IN ('Gegenereerd', 'Actief', 'Tegenboeking')),
    OriginalId   INT               NULL,
    ReasonCode   VARCHAR(50)       NULL,
    CreatedAt    DATETIME2         NOT NULL DEFAULT GETDATE(),
    CreatedBy    NVARCHAR(100)     NOT NULL DEFAULT SUSER_SNAME(),
    CONSTRAINT PK_FbFalAss
        PRIMARY KEY CLUSTERED (Id),
    CONSTRAINT FK_FbFalAss_FbFal
        FOREIGN KEY (FbFalId) REFERENCES dbo.FbFal (Id),
    CONSTRAINT FK_FbFalAss_Original
        FOREIGN KEY (OriginalId) REFERENCES dbo.FbFalAss (Id)
);

13. Helpdocumentatie

13.3 Helptekst tabblad

Gebruik dit tabblad om omzet uit achteraf-abonnementen toe te kennen aan de juiste financiële periode.

  • Start met Genereer periodetoekenningsregels om regels op te bouwen.
  • Gebruik daarna Journaliseer toekenningsregels om te verwerken.
  • Gebruik Journaliseren ongedaan maken alleen om actieve toekenningen terug te draaien.

13.4 Helptekst per actie

ActieHelptekst
GenereerMaakt toekenningsregels aan met status Gegenereerd. Nog geen journaalposten.
JournaliseerBoekt journaalposten voor geselecteerde regels en zet status naar Actief.
Ongedaan makenDraait actieve toekenning terug door tegenboekingsregel aan te maken. Origineel blijft bewaard.

13.5 Statuswaarden

StatusUitleg
GegenereerdDe toekenningsregel is voorbereid maar nog niet geboekt.
ActiefDe toekenningsregel is geboekt en telt mee in de omzettoekenning.
TegenboekingDe actieve toekenningsregel is financieel teruggedraaid.
